# Health Journal Design Principles

These principles guide product, UX and engineering decisions for Health Journal.

## 1. Fast to use

Common tasks should require as few taps and as little typing as possible.

## 2. Clear over clever

Use simple, descriptive language and familiar controls. Avoid decorative or conversational UI text.

## 3. Information first

The application exists to help identify relationships between meals, drinks and symptoms. Prioritise useful information over decoration.

## 4. Consistent

Similar actions should behave consistently throughout the application. Controls, terminology and navigation should be predictable.

## 5. Progressive disclosure

Show the essentials first. Additional detail should be available when needed without cluttering the primary interface.

## 6. Evidence-led development

Every bug should first become an automated test, then be fixed. New features should preserve the existing test suite.

## 7. Mobile-first

The primary experience is on a phone. Desktop support is important, but mobile usability takes priority.

## 8. Calm and factual

The interface should be professional, concise and neutral. Avoid unnecessary greetings, encouragement or conversational language.
